DESIGN DECISIONS:
1. Removed gradient background (linear-gradient → solid gray-50)
   Reason: More professional, modern SaaS aesthetic (follows Stripe/Vercel pattern)

2. Removed confirm password field in signup
   Reason: Single password field reduces friction, modern apps verify server-side

3. Password strength indicator on signup only
   Reason: Guides users to strong passwords at registration, not at login

4. Show/hide password toggle (Eye icon)
   Reason: Improves UX without sacrificing security (password still masked by default)

5. Remember me on login only
   Reason: Convenient for desktop users, standard in modern auth flows

6. Central auth layout without branding in layout.tsx
   Reason: Branding in each page allows flexibility (future social login, verification flows)
