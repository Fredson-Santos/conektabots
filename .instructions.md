# Instruções de Segurança para Agentes

## 🔒 Regras Obrigatórias

Todos os agentes trabalhando neste workspace devem seguir as regras definidas em:
- [AGENT_RULES.md](AGENT_RULES.md) — Resumo rápido
- [.agents/AGENTS.md](.agents/AGENTS.md) — Regras completas

## 🚫 Pastas Protegidas (NUNCA Tocar)

```
.agents/              ← Skills e configurações de agentes
.git/                 ← Histórico de versão
.project/             ← Configurações do projeto
supabase/migrations/  ← Migrações do banco de dados
alembic/versions/     ← Migrações Alembic
tests/                ← Suite de testes
worker/               ← Workers de processamento
.vscode/              ← Configurações de IDE
```

## ⚠️ Antes de Deletar ou Modificar Estrutura

**Verificação obrigatória:**
1. ✅ Confirme que a pasta/arquivo não está na lista protegida
2. ✅ Verifique se o usuário solicitou explicitamente
3. ✅ Se for arquivo sensível (`.env`, `requirements.txt`, etc.), peça confirmação
4. ✅ Nunca toque em `.git/` mesmo que solicitado

## 📋 Operações Seguras

Agentes podem livremente:
- ✅ Criar e editar arquivos em `app/`, `worker/`, `tests/`, `docs/`
- ✅ Criar novos services, routers, schemas
- ✅ Refatorar código existente
- ✅ Adicionar migrações com nomes descritivos
- ✅ Atualizar `tests/`

## 🔐 Segurança de Credenciais

- ❌ NUNCA commitar `.env` ou expor API keys
- ✅ Use `${VARIABLE}` em arquivos de configuração
- ✅ Armazene secrets em `.env` (não versionado)
- ✅ Deixe exemplos em `.env.example`
